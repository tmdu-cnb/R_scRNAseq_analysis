# Log2(FC)分布生成のアルゴリズム説明

## なぜ分布が生成されるのか？

Log2(FC)のRidge Plotで分布が表示される理由は、**ブートストラップサンプリング**と**二項分布**を使ったシミュレーションによるものです。

---

## アルゴリズムの流れ

### ステップ1: 観測データから比率を計算

まず、実際のデータから各細胞タイプの比率を計算します：

```
例：顆粒細胞の場合
- DTA群: 16,792 cells / 27,050 cells = 62.08%
- CTL群: 19,683 cells / 26,288 cells = 74.87%
```

この比率は**点推定値**（1つの値）です。

### ステップ2: 二項分布によるシミュレーション

観測された比率には**不確実性**があります。例えば：
- 「もし別のサンプルを観測したら、比率はどうなるか？」
- 「観測誤差を考慮すると、真の比率はどの範囲にあるか？」

この不確実性を評価するため、**二項分布（Binomial Distribution）**を使ってシミュレーションを行います。

#### 二項分布とは？

二項分布は、以下のような状況をモデル化します：

- **試行回数**: $N$（総細胞数）
- **成功確率**: $p$（細胞タイプの比率）
- **成功回数**: $k$（その細胞タイプの細胞数）

例：顆粒細胞の場合
- DTA群: $N = 27,050$（総細胞数）、$p = 0.6208$（62.08%）
- 「27,050回の試行で、各試行が62.08%の確率で成功する」と仮定

### ステップ3: ブートストラップサンプリング

各細胞タイプについて、**1,000回**のシミュレーションを実行します：

```r
# コードの該当部分
n_bootstrap <- 1000  # 1,000回のシミュレーション

# DTA群のシミュレーション
dta_samples <- rbinom(n_bootstrap, row$DTA_total, row$Proportion_DTA) / row$DTA_total
# → 1,000個の「もし別のサンプルだったら」という比率を生成

# CTL群のシミュレーション
ctl_samples <- rbinom(n_bootstrap, row$CTL_total, row$Proportion_CTL) / row$CTL_total
# → 1,000個の「もし別のサンプルだったら」という比率を生成
```

#### 具体例：顆粒細胞の場合

**DTA群のシミュレーション（1,000回）**：
```
1回目: rbinom(1, 27050, 0.6208) = 16,850 cells → 62.29%
2回目: rbinom(1, 27050, 0.6208) = 16,720 cells → 61.81%
3回目: rbinom(1, 27050, 0.6208) = 16,810 cells → 62.14%
...
1000回目: rbinom(1, 27050, 0.6208) = 16,780 cells → 62.03%
```

**CTL群のシミュレーション（1,000回）**：
```
1回目: rbinom(1, 26288, 0.7487) = 19,650 cells → 74.75%
2回目: rbinom(1, 26288, 0.7487) = 19,720 cells → 75.02%
3回目: rbinom(1, 26288, 0.7487) = 19,680 cells → 74.87%
...
1000回目: rbinom(1, 26288, 0.7487) = 19,700 cells → 74.94%
```

### ステップ4: Log2(FC)の計算

各シミュレーション回について、Log2(FC)を計算します：

```r
# コードの該当部分
fc_samples <- dta_samples / (ctl_samples + 1e-10)  # Fold Change
log2fc_samples <- log2(fc_samples)  # Log2(FC)
```

**具体例**：
```
1回目: Log2(FC) = log2(0.6229 / 0.7475) = log2(0.8333) = -0.263
2回目: Log2(FC) = log2(0.6181 / 0.7502) = log2(0.8240) = -0.279
3回目: Log2(FC) = log2(0.6214 / 0.7487) = log2(0.8300) = -0.268
...
1000回目: Log2(FC) = log2(0.6203 / 0.7494) = log2(0.8277) = -0.273
```

### ステップ5: 分布の可視化

1,000個のLog2(FC)値が生成され、これが**分布**として表示されます。

---

## なぜ分布が生成されるのか？（統計学的な理由）

### ① 観測データの不確実性

実際に観測された比率（例：62.08%）は、**真の比率の推定値**です。もし別のサンプルを観測したら、少し異なる値になる可能性があります。

### ② 二項分布によるモデル化

細胞数のカウントは、以下のようにモデル化できます：

$$\text{細胞数} \sim \text{Binomial}(N, p)$$

ここで：
- $N$ = 総細胞数
- $p$ = 真の比率（未知）

観測された比率 $\hat{p} = \frac{\text{観測された細胞数}}{N}$ を $p$ の推定値として使い、二項分布から新しいサンプルを生成します。

### ③ ブートストラップ法の原理

ブートストラップ法は、**観測データから再サンプリング**することで、統計量の分布を推定する方法です。

- **実際の方法**: 観測データからランダムに再サンプリング
- **このスクリプトの方法**: 観測された比率をパラメータとして、二項分布から新しいサンプルを生成

どちらも、**「もし別のサンプルだったら」**という状況をシミュレーションしています。

---

## 分布の解釈

### 分布の幅（分散）

分布が広い = 不確実性が大きい
- サンプルサイズが小さい場合
- 比率が中間的な値（50%付近）の場合

分布が狭い = 不確実性が小さい
- サンプルサイズが大きい場合
- 比率が極端な値（0%または100%に近い）の場合

### 分布の位置（平均）

分布の中心（平均）が、実際のLog2(FC)値に近い：
$$\text{Log2(FC)} = \log_2\left(\frac{0.6208}{0.7487}\right) = -0.270$$

### 分布と0の関係

- **分布が0より左側**: DTA群で減少傾向
- **分布が0より右側**: DTA群で増加傾向
- **分布が0を含む**: 統計的に有意な差がない可能性

---

## アルゴリズムのまとめ

```
1. 観測データから比率を計算（点推定）
   ↓
2. 二項分布を使って「もし別のサンプルだったら」を1,000回シミュレーション
   ↓
3. 各シミュレーションでLog2(FC)を計算
   ↓
4. 1,000個のLog2(FC)値が分布として生成される
   ↓
5. Ridge Plotで可視化
```

---

## 注意点

### この方法の限界

1. **二項分布の仮定**: 各細胞が独立に分類されると仮定（実際にはクラスター効果などがある可能性）
2. **ブートストラップの仮定**: 観測された比率が真の比率の良い推定値であると仮定
3. **ベイズ統計との違い**: この方法は頻度主義的なアプローチ。ベイズ統計（DMM）とは異なる。

### より正確な方法

より正確な不確実性の評価には、**ベイズ統計（DMM）**を使います：
- ディリクレ分布による階層的モデリング
- ハミルトニアン・モンテカルロ（HMC）による事後分布の推定
- 89%ベイズ信用区間による有意性判定

## コードの該当部分

```r
# ブートストラップサンプリングで分布を生成
set.seed(123)  # 再現性のため
log2fc_samples_list <- list()

for (i in 1:nrow(log2fc_data)) {
  row <- log2fc_data[i, ]
  
  # 各グループで1000回ブートストラップサンプリング
  n_bootstrap <- 1000
  
  # DTA群: 二項分布から1,000個の比率を生成
  dta_samples <- rbinom(n_bootstrap, row$DTA_total, row$Proportion_DTA) / row$DTA_total
  
  # CTL群: 二項分布から1,000個の比率を生成
  ctl_samples <- rbinom(n_bootstrap, row$CTL_total, row$Proportion_CTL) / row$CTL_total
  
  # Fold Changeを計算
  fc_samples <- dta_samples / (ctl_samples + 1e-10)
  
  # Log2(FC)を計算
  log2fc_samples <- log2(fc_samples)
  
  # 結果を保存
  log2fc_samples_list[[i]] <- data.frame(
    CellType = row$CellType,
    Log2FC = log2fc_samples
  )
}

# 全てのサンプルを結合
log2fc_expanded <- bind_rows(log2fc_samples_list)
```

---

## まとめ

**Log2(FC)の分布が生成される理由**：

1. **観測データの不確実性**を評価するため
2. **二項分布**を使って「もし別のサンプルだったら」をシミュレーション
3. **1,000回のシミュレーション**で1,000個のLog2(FC)値を生成
4. これらが**分布**として可視化される

この方法により、単なる点推定値ではなく、**不確実性を考慮した比較**が可能になります。
